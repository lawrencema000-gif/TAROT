## 🧪 Not a bug / test artifact (verified in source, no code change)

| # | Finding | Why it's not a real bug |
|---|---|---|
| 1 | "Global single-letter hotkeys hijack nav" | Grepped all of `src/`: zero `addEventListener('keydown')` outside a single Sheet ESC-close. No `data-hotkey`, no `accessKey`, no hotkey library in `package.json`. The stack trace the agent saw (`vendor-react-e-hhefBg.js:8:59847`) is standard React-Router internals — not a hotkey handler. Most likely Playwright `browser_press_key` misuse or shared-browser cross-talk. |
| 3 | `/ai/quick` + `/ai/tarot` redirect | Both routes are registered in [App.tsx:437-438](../src/App.tsx#L437-L438) and the page components contain no redirect logic. Symptom disappears when session is fresh. |
| 4 | Forgot-password silently fails | `AuthPage.handleResetPassword` correctly calls `supabase.auth.resetPasswordForEmail`; form is `onSubmit` bound with `type="submit"` button. Fires on a clean session. |
| 6 | "Routes silently redirect" | `App.tsx` has explicit routes for every page the agent listed (community, advisors/*, live-rooms/*, sandbox, blog/:slug, tarot-meanings/:slug). Catch-all `*` → `Navigate to="/"` only triggers for genuinely unknown paths. |
| 7 | Session token disappears spontaneously | The 3 parallel agents shared one Playwright browser context. `sb-*-auth-token` is scoped to one origin — writes from Agent B's signin overwrote Agent C's session on every tool call. Same explanation for Agent C's "email mutation" finding — captured email matched Agent B's test account. |
| 12 | Settings auto-opens on `/horoscope` | Not reproducible in isolated session. |
| 14 | AskOracle missing on tarot-meaning pages | Gated on `user` in [TarotCardMeaningPage.tsx:255](../src/pages/TarotCardMeaningPage.tsx#L255). Renders when authenticated. Agent was logged out at that point (see #7). |
| 16 | Onboarding "I already have an account" redirects to blog | No such button/link in [OnboardingPage.tsx](../src/pages/OnboardingPage.tsx) targets blog. Not reproducible. |
| 17 | Onboarding intent "Love" pre-activated when clicking "Career" | Can't find the component. OnboardingPage has 3 steps: language / welcome / create_account — no intent selector. OAuthOnboarding has no intent selector either. Agent may have confused another screen. |
